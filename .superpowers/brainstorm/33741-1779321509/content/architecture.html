Key Architecture Decisions

Expo Router

File-based routing with typed routes. (auth) and (tabs) groups for auth-gated navigation. Deep linking for push notifications.

Clerk → Supabase JWT bridge

Clerk handles OAuth/SAML UI and user profiles. On sign-in, a Clerk JWT is exchanged for a Supabase session via a custom function, enabling RLS.

TanStack Query (Driver) + WatermelonDB (Guard)

Driver app uses React Query for server state with optimistic updates. Guard app adds WatermelonDB for offline permit cache and a sync queue.

Zustand for ephemeral UI state

Scanner animation state, map viewport position, selected filters — nothing that should survive app restart.

External Services

Clerk

Auth UI, user profiles, role claims in JWT metadata

Supabase

PostgreSQL DB, RLS, real-time subscriptions, Edge Functions

Expo

EAS Build for dev/prod builds, Expo Go for quick iteration
