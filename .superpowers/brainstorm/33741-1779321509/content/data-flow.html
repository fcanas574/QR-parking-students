Real-time Subscriptions

lot_occupancy changes

Both apps subscribe to occupancy updates via Supabase Realtime. Map markers and capacity bars update live without polling.

Scan-to-display latency

Guard scans → local WatermelonDB insert → optimistic UI update → sync queue → Supabase → occupancy trigger fires → Realtime broadcast → all apps update.

Auth Token Flow

  1. Clerk.signIn() → Clerk session + JWT
  2. Clerk JWT sent to Supabase Edge Function "exchange-token"
  3. Edge Function validates Clerk JWT, extracts role from metadata
  4. Edge Function calls supabase.auth.admin.generateLink() → Supabase session
  5. Client receives Supabase access_token + refresh_token
  6. All subsequent Supabase calls use this session → RLS policies apply

  On token expiry: Clerk refresh → re-run exchange silently
