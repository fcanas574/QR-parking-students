Data Model — Supabase Schema

Core tables with Row Level Security enabled, Clerk user IDs as the foreign key anchor

Users & Profiles

profiles                         ← created on first Clerk sign-in via trigger
├── id: uuid (PK)
├── clerk_id: text (UNIQUE)         ← links to Clerk user
├── full_name: text
├── role: enum(student, staff, security, admin, super_admin)
├── email: text
├── phone: text
├── avatar_url: text
└── created_at: timestamptz

vehicles
├── id: uuid (PK)
├── profile_id: uuid (FK → profiles)
├── plate_number: text (UNIQUE)
├── make: text
├── model: text
├── color: text
├── is_verified: boolean
└── created_at: timestamptz

Parking Lots & Permits

parking_lots
├── id: uuid (PK)
├── name: text                    ← "Lot A", "Lot C", "Visitor Lot V"
├── total_spaces: int
├── location: geography(POINT)    ← for map display
├── is_active: boolean
└── created_at: timestamptz

permits
├── id: uuid (PK)
├── profile_id: uuid (FK → profiles)
├── vehicle_id: uuid (FK → vehicles)
├── lot_id: uuid (FK → parking_lots)
├── status: enum(active, expired, suspended, pending_approval)
├── starts_at: timestamptz
├── expires_at: timestamptz
├── payment_status: enum(paid, unpaid, waived)
├── payment_id: text              ← external payment ref (stubbed)
├── auto_renew: boolean
└── created_at: timestamptz

Access Logs & Capacities

access_logs
├── id: uuid (PK)
├── permit_id: uuid (FK → permits, nullable)
├── vehicle_id: uuid (FK → vehicles)
├── lot_id: uuid (FK → parking_lots)
├── scanned_by: uuid (FK → profiles)  ← security guard who scanned
├── direction: enum(entry, exit)
├── method: enum(nfc, qr, manual)
├── is_valid: boolean              ← was permit active at scan time?
├── scanned_at: timestamptz
└── synced_at: timestamptz         ← NULL until offline sync completes

lot_occupancy                   ← materialized-ish: updated by trigger on access_logs
├── id: uuid (PK)
├── lot_id: uuid (FK → parking_lots)
├── current_count: int
├── capacity: int
├── last_updated: timestamptz
└── CONSTRAINT one_row_per_lot UNIQUE (lot_id)

RLS Policy Summary

Students & Staff

Read own profile and permits. Insert/update own vehicles. Read all parking_lots. Read own access_logs. Read lot_occupancy.

Security

Read all profiles (plate lookup). Insert access_logs (entrance/exit). Read all permits (validation). Read/write lot_occupancy.

Admin & Super Admin

Full CRUD on permits, parking_lots, vehicles. Read all access_logs. Approve/reject permits. Web app only (not mobile).
