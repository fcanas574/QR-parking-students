Offline Strategy & Error Handling

Guard app resilience and error states across both apps

Guard App — Offline Scanning

INITIAL SYNC (at sign-in)
   ├── Pull all active permits → WatermelonDB
   ├── Pull all parking_lots → WatermelonDB
   ├── Pull last 7 days of access_logs → for reference
   └── Store sync timestamp

SCAN FLOW (offline)
   ├── Guard taps "Scan" or phone detects NFC tag
   ├── Read plate/vehicle ID from NFC/QR
   ├── Lookup vehicle in local WatermelonDB → get associated permit
   ├── Validate: permit active? correct lot? not already inside?
   ├── Show result: ✓ VALID or ✗ DENIED with reason
   ├── Insert access_log row locally (synced_at = NULL)
   ├── Increment local lot_occupancy count
   └── Push to sync queue

SYNC ENGINE (on connectivity restored)
   ├── 1. Push queued access_logs (oldest first)
   ├── 2. On conflict (two guards scanned same vehicle):
   │     timestamp wins, later scan gets warning
   ├── 3. Pull fresh permits (new purchases, expirations)
   ├── 4. Pull current lot_occupancy from server
   ├── 5. Reconcile local counts with server
   └── 6. Update UI

CONNECTIVITY INDICATOR
   ├── Green dot + "Online" — live connection
   ├── Amber dot + "Offline · 12 queued" — syncing later
   └── Red dot + "Sync failed · 3 pending" — manual retry needed

Error States — Both Apps

Network Errors

Driver: TanStack Query retries 3x with exponential backoff, shows skeleton + "Can't reach server" toast after final failure.
Guard: Falls back to WatermelonDB. Banner shows sync queue count. No data loss.

Auth Errors

Clerk session expired → redirect to sign-in. Supabase token expired → silent re-exchange via Edge Function. Role mismatch → "This account doesn't have access to this app" screen with sign-out button.

Scan Errors

Invalid QR/NFC: Red ring animation + "Invalid tag. Try again or enter manually."
Expired permit: Yellow ring + "Permit expired — {date}. Contact parking office."
Wrong lot: "Vehicle XYZ-9876 not authorized for Lot C. Valid for: Lot B."
Duplicate scan: "Vehicle already logged as {entry/exit}. Override?"

Payment Errors (Driver)

Payment module stubbed → "Payment processing coming soon" screen. Payment declined → "Unable to process. Try again or contact parking office." No permit created until payment confirmed.

Edge Cases

Concurrent scans

Two guards scan same vehicle at different gates. Server resolves by timestamp. Both see confirmation. Occupancy reconciled server-side.

Ghost occupancy

Vehicle exits without scan (manual override, hardware failure). Lot occupancy drifts. Admin can manually correct counts in web dashboard.

Multi-vehicle permit

One permit, multiple registered vehicles? One vehicle active at a time. Scanning a second vehicle while first is "inside" → denied.

Plate change

Student changes plates → must re-verify vehicle. Old plate invalidated. New plate linked to existing permit after verification.
